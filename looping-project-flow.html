Looping

Proximity Based Social Networking App

Platform: Flutter (iOS + Android) Budget: INR 2,50,000 Timeline: 14 Weeks Date: 15 March 2026

Looping is a proximity based social networking app that allows users to discover, connect and communicate with verified people nearby using Bluetooth Low Energy (BLE) and location services. Users can chat, make voice calls, video calls and share verified profiles in a privacy first environment.

1. Technology Stack

Layer Technology Purpose
Frontend Flutter 3.x (Dart) Cross platform mobile app for iOS and Android from a single codebase
Backend Node.js (TypeScript) REST API server, business logic, call signaling, push notification dispatch
Database PostgreSQL Primary relational database for users, connections, chats, calls, reports
Cloud Hosting E2E Networks Indian cloud provider for server hosting, storage and networking
Email Service Mailgun Transactional emails for OTP, verification, notifications and password resets
Voice and Video Agora / Twilio WebRTC powered 1:1 voice and video calling with low latency
Push Notifications Firebase Cloud Messaging (FCM) Push notifications for messages, calls and connection requests
Identity Verification DigiLocker API Government API for Aadhaar and PAN verification without storing raw IDs
Proximity Discovery BLE + PostGIS Bluetooth Low Energy scanning with geo query fallback for nearby discovery
Authentication JWT + OTP Phone OTP verification, email/password login, token based session management

3. Module Flows

Module 01: Authentication and Registration

Open App → Login Screen → Phone OTP / Email Login → Step 1: Profile Setup → Step 2: Contact and ID → Step 3: Purpose → Step 4: Social and Docs → Step 5: Privacy and Terms → Home Screen

Module 02: Nearby Discovery

Tap Find Nearby → BLE Scan Starts → Radar Animation → Users Appear as Bubbles → Tap User Bubble → Mini Profile Card

Module 03: Connection System

Send Request → Push Notification → Receiver Reviews → Accept / Decline → Connected → Chat and Calls Unlocked

Module 04: Real Time Chat

Open Chat → Type Message → Send → Delivered (Double Tick) → Read (Blue Ticks)

Module 05: Voice Calling

Tap Call Button → FCM Signal Sent → Receiver Phone Rings → Accept Call → WebRTC Connected → Active Call (Timer + Controls) → End Call

Module 06: Video Calling

Tap Video Call → FCM Signal Sent → Incoming Screen (Dark UI) → Accept → Active Video (Remote + PiP) → Controls (Flip, Mute, End) → End Call

Module 07: Screenshot Protection

Open Guard Settings → Toggle Protections → Screenshot Blocked (Android)
Open Guard Settings → Toggle Protections → Screenshot Detected (iOS) → Other User Notified

Module 08: Settings and Deployment

Settings Screen → Account / Privacy / Notifications → Block and Report → Delete Account
Feature Complete → Device Testing (7+ devices) → Bug Fixes → Play Store + App Store Submission

8. Architecture Overview

Flutter App (iOS + Android)
    |
    |--- REST API (HTTPS)
    |
Node.js TypeScript Backend (E2E Networks)
    |
    |--- PostgreSQL (Primary Database)
    |--- Mailgun (Email OTP, Verification, Notifications)
    |--- FCM (Push Notifications)
    |--- Agora / Twilio (Voice and Video WebRTC)
    |--- DigiLocker API (Identity Verification)
    |--- E2E Object Storage (Profile Photos, Media)

Database Tables (PostgreSQL)

  • users - Profile data, auth credentials, privacy settings, verification status
  • connections - User pairs, status (pending/accepted/rejected), timestamps
  • messages - Chat messages with sender, receiver, content, status, timestamps
  • calls - Call records with type, duration, status, caller and callee
  • reports - User reports with reason, description, timestamps
  • blocked_users - Block relationships between users
  • devices - FCM tokens and device info for push notifications
  • social_links - OAuth linked accounts (Google, Instagram, Facebook)

9. Legal Compliance

  • Aadhaar Act 2016: No raw Aadhaar numbers stored. Only masked display and verification status.
  • DPDPA 2023: Consent collection, right to erasure, data minimization, Grievance Officer details in app.
  • IT Act 2000: Reasonable security practices, encrypted data at rest (AES 256) and in transit (TLS 1.3).
  • All user data stored on E2E Networks (India based servers).
  • Privacy Policy and Terms of Service reviewed by legal counsel before launch.
  • Account deletion purges all personal data within 30 days.
